단위처리
테이블 A, B가 있을 때 두 테이블을 묶어 처리하는 것.
A에 데이터를 삽입하고, B에 삽입하기 전에 예외가 발생하면 B에는 데이터가 안들어감 -> 단위처리 실패 ->  rollback
둘 다 성공하면 commit

force: 뷰를 만들 때, 원본이 없어도 뷰를 만드는 옵션.

뷰에 데이터를 저장하면 원본 테이블에 저장됨.

오라클 옵티마이저 - 성능 개선을 위한 툴.

트리거 
방아쇠, 연쇄반응 등의 사전적인 의미를 가지고 있는데, 오라클에서도 한마디로
방아쇠를 당기면 총알이 발사가 되듯이 특정 테이블이 변경되면 감지했다가 설정한 다른
테이블에 자동으로 변경되도록 하기 위해서 사용한다. 자바의 리스너라고 생각해도 된다.
* 트리거는 데이터베이스가 미리 정해 놓은 조건을 만족하거나 어떤 동작이 수행되면
자동으로 수행하는 동작을 말한다
* 트리거는 수동으로는 실행 시킬수 없다. 특정 동작을 이벤트로 그로 인해서만 실행되는 프로시저의 일종이다.
* 데이터 흐름 제어명령(TCL)은 사용할 수 없다.!
-- 목적 
1. 데이터 무결성 구현 ( 제한조건으로 구현할 수 없는 경우)
2. 업무 자동화(연금계산, 급여계산,.. 회계관리 등 연쇄작용하는 업무에 많이 사용된다)
3. 보안(감사)성 강화하는데 사용(이벤트를 발생한 유저 기록 관리)
4. 데이터 복제 구현
