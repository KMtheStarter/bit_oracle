ROWID: DBMS가 스스로 지정함.

- 오라클 자료형
CLOB: 속성을 가지고 있는 데이터 저장 가능.
VARCHAR2 와 CHAR의 차이는 가변인가 아닌가의 차이. 하지만 오버플로우는 둘 다 허용 안됨.

제약조건: 데이터의 정확성과 일관성을 보장하기 위해 테이블 생성 시 각 컬럼에 대해 정의 하는 규칙.
- CHECK: 해당 컬럼에 저장 가능한 데이터 값의 범위나 조건 지정. 물론 서버나 js를 통해 값을 거를 순 있지만,
데이터 병합등을 고려하여 구축.

제약조건의 이름을 설정해야 오류를 찾기가 쉬움.
- 이름이 없다면 SYS0001 이런 방식으로 오류를 알려줌.
- 그러면 user_constraint라는 테이블에 접근하여 해당 코드가 무슨 오류인지 확인해야함.
- 제약조건의 이름이 설정되어 있다면 해당 이름을 언급하며 오류를 알려줌.

lsnrctl: 오라클 리스너 호출 > service / status 를 입력하면 SID 확인 가능.

select * from tab; > 모든 테이블 조회
purge recyclebin; > 휴지통 비우기
col tname format a10 > 테이블 예쁘게 나옴

트리거를 쓰면 편리하지만, 자원을 많이 잡아먹기 때문에 오라클의 성능이 저하됨.
테이블을 자동으로 create할 때 트리거는 넣지 않는 것이 좋음.

제약조건을 명시하지 않으면 SYS_Cn의 형태로 자동으로 들어간다.
